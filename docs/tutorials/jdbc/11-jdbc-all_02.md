## 1. 概要

前回のチュートリアルではJDBCを利用してデータベースに接続してレコードを習得するというプログラムを題材にして、Javaからデータベースに接続するときに必要な知識について説明しました。  
しかし、データベース接続関連オブジェクトはプログラムが終了するまで破棄されずに残ってしまい、最終的にメモリリークを起こしてプログラムが強制終了してしまうという問題点があります。  
言い方を変えれば、この問題はメモリリークを起こすリスクを孕んでいるということになります。

このチュートリアルでは、データベース接続オブジェクトによるメモリリークのリスクを回避する方法を解説します。

### 手順-8：データベース接続関連オブジェクトのクローズ処理
```java
} finally {
  // 手順-8. データベース接続関連のオブジェクトのクローズ処理
  try {
    if (rs != null) {
      rs.close();
    }
    if (pstmt != null) {
      pstmt.close();
    }
    if (conn != null) {
      conn.close();
    }
  } catch (SQLException e) {
    // 例外が発生した場合：スタックトレースを表示（必要最低限のエラー情報を表示）
    e.printStackTrace();
  }
}
```
- データベース接続関連オブジェクトをクローズするのは、例外発生の有無に関係なく必要な処理なので `finally` ブロックに記述します。
- データベース接続関連オブジェクトのクローズの順序は **取得された順と逆** になります。  
取得した順と逆で閉じることで、依存関係に問題が生じないようにしています。
