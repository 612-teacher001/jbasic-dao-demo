## 1. 概要
このチュートリアルでは、DAOを利用して productsテーブルの **商品名にキーワードを含むレコードを検索し表示する方法** を説明します。

DAO（Data Access Object）パターンを導入することで、データベース操作を専用クラスにまとめ、呼び出し側では「検索して結果を受け取るだけ」というシンプルなコードを書けるようになります。

## 3. 手順ごとの解説

ここで解説する手順は `jp.example.app.t2.DaoKeyword#main(String[])` メソッドのコードを前提としています。  
実際にコードを確認しながら読み進めてください。

👉 DAOの宣言・インスタンス化、リスト表示、DAOクローズの部分は前回のチュートリアル（[全件検索](./21-dao-all.md)）と同じ流れです。  
ここでは **新しく登場する処理**（キーワード入力と検索処理）に絞って説明します。

### 手順-1：DAOを宣言する
- 前回の処理と変更はありません。  

### 手順-2：DAOをインスタンス化する
- 前回の処理と変更はありません。  

### 手順-3：キーボードから入力された検索キーワードを取得する
``` java
String keyword = Keyboard.getInputString("検索キーワードを入力してください：");
```
- キーボードから入力された文字列を検索キーワードとして取得します。  
また、簡略化のため入力値チェックは省略しています。

### 手順-4：商品名にキーワードを含む商品のリストを取得する
```java
List<Product> list = dao.findByNameLikeKeyword(keyword);
```
- `ProductDAO#findByNameLikeKeyword(String)` メソッドを呼び出すことで、商品名にキーワードを含む商品の一覧を取得します。  
取得されたリストは 主キー(`id` フィールド)の昇順で並べ替えられています。
- 一致する商品が存在しない場合は空リストが返ります。

### 手順-5：商品リストを表示する
- 前回の処理と変更はありません。  

### 手順-6：DAOをクローズする
- 前回の処理と変更はありません。

## 4. まとめ

ここのコードで学ぶべきポイント：

  - DAO クラスにより、SQLやJDBCリソース管理を意識せずにシンプルな検索処理を記述できる
  - 商品名の **部分一致検索** は ProductDAO#findByNameLikeKeyword(String) を呼ぶだけで実現できる
  - 一覧系の処理では、（手順-0 で行ったように）検索対象一覧を事前に表示しておくことで、検索対象をイメージしやすくなる。  
また、結果リストを整形して出力することで、実行結果が分かりやすくなる。
